# 核心要点

eFuse（电子熔丝）是物理上的一次性编程单元，通过高电压永久烧断熔丝，将比特从 0 变为 1。一旦完成，不可逆转。

## eFuse 存储的关键安全信息

- 安全启动密钥的哈希值（验证固件签名）
- 生命周期状态（Lifecycle）：出厂 → 开发 → 量产 → 现场部署，逐级收紧
- JTAG/SWD 永久禁止位
- 安全配置字（禁用启动模式、强制加密、锁定内存区域）

## 三大不可逆关卡

### 关卡一：安全启动（Secure Boot）
- BootROM 读取 eFuse 公钥哈希 → 验证固件签名 → 失败则静默死锁
- **灾难场景**：eFuse 公钥哈希误烧为错误值 → 正确固件永远被拒
- **灾难场景**：安全版本计数器只增不减 → 固件版本低于计数器则拒绝启动

### 关卡二：生命周期（Lifecycle）
- 开发阶段：JTAG/SWD 可用，eFuse 大部分可编程
- 量产阶段：调试口永久锁死，启动源受限，关键 eFuse 写保护
- **单向门**：开发 → 量产不可逆

### 关卡三：调试授权
- eFuse 控制 JTAG/SWD 使能：烧断即永久禁止
- 或通过密码哈希的挑战-响应协议临时解锁

## Bringup 铁律

1. **只读安全状态，绝不执行不可逆 eFuse 编程**
2. 启动早期读取生命周期、检查安全启动状态、确认调试授权
3. 用 `BRINGUP_PHASE` 宏包裹所有 eFuse 烧写代码路径
4. 调试口保持临时开放，不永久烧写调试使能位

## EL1/EL3 交互

- Bringup 代码运行在 EL1（非安全世界），安全模块在 EL3
- 通过 SMC（安全监控调用）和共享内存与 TF-A 安全固件交互
- 绝不绕过安全监视器直接访问安全物理地址

## 正例 vs 反例

❌ 反例：直接烧写调试禁止位，不检查生命周期，无逃生通道
✅ 正例：先读生命周期 → 开发模式确保调试开放 → 量产模式仅认证解锁 → Bringup 阶段跳过所有 eFuse 永久编程

## 一句话总结

> 安全模块是芯片启动时第一个说话的神。你必须先聆听它，再顺应它，绝不可在 Bringup 的混乱中违逆它。
