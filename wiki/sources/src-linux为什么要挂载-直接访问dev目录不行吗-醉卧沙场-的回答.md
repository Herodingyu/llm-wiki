## Summary

本文深入解答了"为什么Linux需要挂载，不能直接访问/dev目录"这一常见问题。文章通过仓库与文件柜的类比，阐明了/dev目录只是设备的"登记册"（记录设备存在但不展示内容），而挂载是将设备内容映射到文件系统的关键操作。核心在于理解存储设备和文件系统的本质区别：/dev下的设备文件面向的是存储块（扇区），而用户需要的是文件系统提供的文件/文件夹抽象。挂载操作将具体的存储设备与文件系统驱动关联，将文件和目录关系挂载到全局目录树，形成"激活运行状态"。

## Key Points

### 1. 存储设备 vs 文件系统
- **存储设备**：提供块（扇区）访问，如/dev/sda1
- **文件系统**：在存储设备之上提供文件/文件夹抽象
- **关键区别**：直接读取/dev得到的是原始二进制块数据，而非有意义的文件

### 2. /dev 目录的本质
- **设备登记册**：记录所有设备的"身份证"
- **接口文件**：代表硬件本身，不是文件系统
- **直接访问**：可用dd等工具读写原始数据，但无法直接cd进入

### 3. 挂载的核心作用
- **建立关联**：将存储设备上的具体文件系统与内核文件系统驱动关联
- **激活状态**：形成"激活运行状态"的文件系统
- **融入目录树**：将设备内容挂载到全局目录树的某个节点
- **用户可见**：之后才能看到文件和文件夹，并通过文件系统方法访问

### 4. 为什么不能直接访问/dev？
- `/dev/sda1`是设备文件，不是目录
- 设备文件面向存储块，用户面向的是文件抽象
- 文件的数据和元数据可能散落在存储设备的各个角落
- 直接读取存储块如同"大海捞针"

### 5. 挂载命令示例
```bash
mkdir /new_disk
mount /dev/sdc /new_disk
# 之后 /new_disk 下就能看到完整文件系统结构
```

### 6. 现代Linux的自动化
- 桌面发行版（Ubuntu等）自动识别U盘并挂载
- 自动创建挂载点（如`/media/username/USB_DISK`）

### 7. 进阶应用
- **网络存储挂载**：NFS/SMB远程目录挂载到本地
- **内存盘挂载**：tmpfs将内存作为高速磁盘使用
- **绑定挂载**：将已有目录挂载到另一个位置

## Key Quotes

> "/dev/下的设备文件面向的是设备本身，你虽然可以打开、读取、写入一个存储设备，但是你面向的终究是一个存储设备，不是文件系统。"

> "挂载的最主要作用其实就是将一个具体存储设备上的具体文件系统和操作系统中对应的文件系统驱动关联起来。"

> "用户需要面向的单位不是存储块本身，用户面向的单位是'文件'这样的抽象概念。"
