## Summary

本文从嵌入式开发角度通俗解释了文件系统和根文件系统的概念。文件系统是管理存储单元（硬盘、NAND Flash 等）的程序，通过目录/文件形式屏蔽底层读写细节，提供统一的 read/write 接口。挂载是将存储分区关联到目录的过程，删除文件只是断开映射关系而非真正删除数据。根文件系统是挂载到 "/" 根节点的文件系统，存放系统启动必需的命令、配置文件和启动脚本。文章还通过计算机存储发展史（穿孔卡片、纸带、磁带、磁盘）生动展示了文件系统概念的演进。

## Key Points

### 1. 文件系统本质
- 管理存储单元的程序（如硬盘、NAND Flash）
- 将存储空间划分成块，编号，统一管理
- 通过目录/文件形式管理数据
- 屏蔽底层读写时序细节
- 提供统一的 read/write 接口

### 2. 文件系统的价值
| 无文件系统 | 有文件系统 |
|-----------|-----------|
| 需了解 NAND 读写时序 | 通过文件名即可读写 |
| 一页一页读取物理地址 | 通过目录/文件名访问 |
| 不同设备读写方式不同 | 统一接口，设备无关 |

### 3. 挂载（Mount）
- 将存储分区关联到目录
- 目录成为存储分区的访问入口
- 读写目录 = 读写对应分区
- **删除文件 = 断开映射关系（数据仍在，可恢复）**

### 4. 根文件系统
| 属性 | 说明 |
|------|------|
| 挂载点 | "/" 根节点 |
| 内容 | 基本 Linux 命令、系统配置文件、启动脚本 |
| 特点 | 系统启动必需 |
| 嵌入式实现 | 通常是 NAND 的一个分区 |

### 5. 文件系统类型
| 类型 | 特点 | 应用场景 |
|------|------|---------|
| RAM 文件系统 | 存在于内存中，速度快 | 系统启动运行时 |
| 存储文件系统 | 管理磁盘、U盘、SD 卡等 | 持久化数据存储 |

## Evidence

- 无文件系统时需了解 NAND 读写时序，一页一页读取
- 有文件系统后通过文件名即可读写
- 删除文件只是断开映射，数据可恢复（除非格式化）
