- SDRAM具有同步接口

### 12.3.5 DDR3 SDRAM

DDR3进一步增加了预取长度为8以提高设备传输速率。同时还：

- 增加了基于温度的自刷新模式来降低刷新频率
- 增加了编程能力，可以只刷新部分保存有数据的行而不是所有行

## 12.4 高带宽路径

### 12.4.1 Direct RDRAM

其使用了特别的定制的信号协议Rambus Signaling Level (RSL)，是一种high-speed, singled-ended, multidrop, bidirectional bus信号协议。下图12.23展示了其特性，可以在1-1.8V间摆动。

![图片](data:image/svg+xml,%3C%3Fxml version='1.0' encoding='UTF-8'%3F%3E%3Csvg width='1px' height='1px' viewBox='0 0 1 1' version='1.1' xmlns='http://www.w3.org/2000/svg' xmlns:xlink='http://www.w3.org/1999/xlink'%3E%3Ctitle%3E%3C/title%3E%3Cg stroke='none' stroke-width='1' fill='none' fill-rule='evenodd' fill-opacity='0'%3E%3Cg transform='translate(-249.000000, -126.000000)' fill='%23FFFFFF'%3E%3Crect x='249' y='126' width='1' height='1'%3E%3C/rect%3E%3C/g%3E%3C/g%3E%3C/svg%3E)

相比于传统DRAM，Direct RDRAM的典型特性包括：

- path-matched address
- command and data bus topology
- separate row and column address channels with encoded command packets
- separate data channel
- multiple cycle
- packet-based command and address

下图12.24展示了一个128Mb 32bank的Direct RDRAM设备结构，每个Bank分为上下两半，相邻bank共享一个感应放大器。

![图片](data:image/svg+xml,%3C%3Fxml version='1.0' encoding='UTF-8'%3F%3E%3Csvg width='1px' height='1px' viewBox='0 0 1 1' version='1.1' xmlns='http://www.w3.org/2000/svg' xmlns:xlink='http://www.w3.org/1999/xlink'%3E%3Ctitle%3E%3C/title%3E%3Cg stroke='none' stroke-width='1' fill='none' fill-rule='evenodd' fill-opacity='0'%3E%3Cg transform='translate(-249.000000, -126.000000)' fill='%23FFFFFF'%3E%3Crect x='249' y='126' width='1' height='1'%3E%3C/rect%3E%3C/g%3E%3C/g%3E%3C/svg%3E)

相比于DDR高度依赖于宽数据总线而言，Direct RDRAM只使用16-bit数据总线，并且使用Packet格式传输。

为了达到高速信号系统的优势，还专门为其设计了如下图12.25所示的拓扑结构，所有设备都通过同样宽度的数据总线进行连接，每个设备中包含自己独立的译码逻辑。

## 13.2 Row-Buffer-Management Policy

在当代DRAM设备中，感应放大器阵列可以当做时buffer来提供数据的临时存储，行缓存管理策略指的则是用来管理感应放大器的策略。

### 13.2.1 open-page行缓存管理策略

DRAM设备的数据访问需要行激活和列激活两个阶段，当内存访问序列具有高度的时间/空间局部性时，就可以将相邻访问的数据导向到同一行来最大化利用局部性。open-page则旨在保持感应放大器处于打开状态来支持内存访问同一row。那么下一次访问如果是访问同一行的话就可以以最快的速度连续访问。

这样的好处是，如果下一次访问时同一行，则只需要列激活命令就可以很快拿到数据。缺点是，如果下一次访问的是不同行，那么就需要进行一次precharge操作。

### 13.2.2 close-page行缓存管理策略

正好相反，close-page设计用于支持DRAM的随机访问（更多被部署在处理器数量较多的场景）。

### 13.2.3 混合（动态）行缓存管理策略

现代DRAM设备很多可以利用访问历史记录等方式来动态的控制管理策略。

## 13.3 地址映射（翻译）

地址映射方案同样可以直接影响DRAM内存系统的性能，其决定了如何将物理地址解析为channel/rank/bank/row/column ID，目的是尽可能减少bank冲突的可能性并最大化内存系统的并行性，并通常与行缓存管理方案结合使用，然而不同的是，地址映射方案并不能像行缓存管理策略那样动态的调度改变。

下图13.2展示了传统系统架构和新兴的脉冲内存系统结构图：

- 传统系统架构中，处理器工作在虚拟地址，TLB负责虚拟地址到物理地址的映射，而不关系DRAM内存系统的映射方案
- 脉冲内存控制器允许系统使用部分地址空间作为 `shadow address` ，其虚拟地址到物理地址的映射中可以结合MC实际情况，动态映射以减少冲突

### 13.3.1 内存系统组织中的可用并行性

可用并行性层次包括：

- Channel
- 通常由不同的MC独立控制，所以他们之间的使用没有约束
	- 处于性能目的，连续的内存访问通常会被映射到不同的channel
- Rank
- 共享地址/命令/数据总线
	- 由于Rank的切换开销，通常限制将连续DRAM请求发送到不同的Rank
- Bank
- 通常，将连续请求调度到同一Rank的不同Bank比调度到不同Rank更高效
	- 列写命令紧跟列读命令，在不同Rank上执行效率要高于在同一Rank
	- 通常Bank地址映射到地址低位，以尽可能利用读命令的空间局部性，提高并发
- Row
- 同一Bank中通常只能由一个Row处于激活状态，所以同一bank的不同row需要串行访问
	- 所以row地址通常映射到内存地址的高位以减少对不同row的访问
- Column
- 在open-page系统中，column地址通常映射到地址低位
	- 在close-page中（期望将连续地址访问分不到不同bank的不同row），则column地址通常分为两段，低位拿取物理地址的低位，高位则拿取物理地址的高位

### 13.3.3 基本地址映射方案

**open-page基本地址映射方案**

为了性能，相邻的cache line地址会分配到不同channel，然后被映射到不同rank，同一bank和同一row。其对应的基本方案是r:l:b:n:k:z（小写字母代表对应id的bit排布顺序）。

**close-page基本地址映射方案**

连续的cacheline地址同样会被map到不同的channel，但是会映射到不同rank，不同bank。对应的基本方案是r:n:l:b:k:z。

### 13.3.4 并行性vs.扩展能力

为了支持用户购买额外的DRAM来扩容，通常会考虑将channel映射到地址高位，对应的open-page/close-page的映射方案可以是k:l:r:b:n:z或k:l:r:n: b:z。
