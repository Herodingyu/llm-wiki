## 核心主题

外设子系统是 SoC 连接真实世界的入口，其设计目标是在"高速确定的片上系统"与"低速不确定的外部世界"之间建立一套秩序。

## 关键内容

### 一、SoC 为什么需要外设？
芯片完成任务需要三类能力：计算、存储、交互。前两类发生在芯片内部，第三类（交互能力）落在外设子系统身上。外设不是 SoC 的"附属品"，而是面向应用场景的能力集合。

### 二、外部世界很慢，系统内部很快
SoC 内部追求高带宽低延迟，但外部设备低速、异步、不确定。这带来四个问题：
- CPU 不能一直等外设
- 外设数据到来时间不确定，需响应异步事件
- 不同外设速度不同，需缓冲、握手和协议转换
- 外设访问内存时要避免破坏一致性和安全边界

### 三、外设子系统的四大组成
| 组件 | 说明 |
|------|------|
| 外设控制器 | UART/SPI/I2C/USB/Ethernet/GPIO/Timer/PWM 等，负责把软件配置转成引脚波形 |
| 寄存器接口 | CPU 通过读写寄存器控制外设，寄存器是软硬件契约 |
| 中断与事件机制 | 外设主动通知 CPU，避免轮询浪费 |
| DMA 与缓冲 | 高吞吐外设通过 DMA 直接搬运数据，CPU 只做决策 |

### 四、外设为何挂在低速总线上
外设寄存器访问频率低，低速总线（如 APB）面积更小、功耗更低、时序更容易收敛，且能把外设从高性能主干隔离出来。

### 五、从按键点亮 LED 看完整链路
包含：GPIO 采样 → 同步去抖 → 触发中断 → CPU 执行 ISR → 配置 GPIO 输出 → 驱动 LED。涉及引脚复用、寄存器访问、外设总线、中断控制、时钟域同步。

### 六、外设设计的三个关键问题
1. **软件如何控制硬件？** — 寄存器模型必须清晰稳定
2. **数据如何高效流动？** — 低速外设 CPU 直接读写，高速外设必须 FIFO + DMA
3. **外设如何不影响系统稳定性？** — 超时、错误状态、复位、权限隔离、调试可观测性

### 七、架构师视角的设计原则
四句话总结：
- 慢设备不要拖慢快系统（桥接、FIFO、DMA 解耦）
- CPU 不做无意义等待（中断代替轮询，DMA 代替搬运）
- 寄存器就是软硬件契约（稳定、清晰、可验证）
- 外设面对真实世界，必须设计异常路径
