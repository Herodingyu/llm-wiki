## Summary

本文是知乎高赞回答（130赞同），解答了一个常见的内存升级困惑：为什么单独使用时都能达到3200MHz的两根内存条，混插后却只能运行在2933MHz。核心原因在于内存条的SPD（Serial Presence Detect）参数不匹配。每根内存条都内置了EEPROM芯片，存储着不同频率下的时序参数。当两根内存条混插时，BIOS/UEFI需要找到一组双方都能满足的SPD参数才能稳定运行。如果两根条子在3200MHz频率点的时序参数不同，BIOS可能会选择其中较宽松（较低）的那组参数，然后检查另一根内存在哪个频率点能满足这组参数，最终可能导致系统降频到2933MHz运行。文章还提供了在Linux系统下通过i2c-tools和decode-dimms命令读取SPD信息的实用方法，帮助用户自行诊断内存参数。这提醒我们在升级内存时，不仅要注意品牌和频率，更要确保SPD参数的兼容性，最好购买同批次、同型号的产品。

## Key Points

### 1. 问题现象
- 内存A单独使用：3200MHz正常
- 内存B单独使用：3200MHz正常
- A+B混插：显示2933MHz
- 品牌相同、频率标称相同

### 2. SPD参数不匹配原理
- **SPD是什么**：Serial Presence Detect，内存条上的EEPROM芯片
- **存储内容**：不同频率下的时序参数（CL、tRCD、tRP、tRAS等）
- **BIOS策略**：必须找到双方都能满足的参数组合

### 3. 频率匹配算法推测

| 步骤 | 操作 | 结果 |
|------|------|------|
| 1 | 比较两根内存在3200MHz的SPD数据 | 发现不匹配 |
| 2 | 选择较宽松（较低）的那组参数 | 作为候选 |
| 3 | 检查另一根内存在哪个频率满足候选参数 | 2933MHz满足 |
| 4 | 最终运行频率 | 2933MHz（降频运行） |

### 4. 其他可能原因
- **批次差异**：不同批次颗粒体质不同，SPD参数微调
- **生产时间**：早批次和晚批次的工艺优化差异
- **颗粒来源**：即使是同品牌，颗粒可能来自不同厂商
- **XMP/EXPO**：超频配置文件的兼容性差异

### 5. 解决方案
1. **最佳方案**：购买同型号、同批次的内存条
2. **手动设置**：在BIOS中手动设置时序参数（需技术基础）
3. **SPD读取**：使用CPU-Z或decode-dimms查看具体参数差异
4. **退换货**：如果差异过大，考虑更换为匹配的内存

### 6. Linux下诊断工具
```bash
# 安装i2c-tools
dnf install i2c-tools -y

# 加载内核模块
modprobe eeprom
modprobe at24
modprobe i2c-i801
modprobe i2c-amd-mp2-pci
modprobe ee1004

# 解码SPD信息
decode-dimms
```

## Key Quotes

> "两个内存条在3200频率点的SPD参数不匹配。而其中一个内存条在2933频率点的SPD参数能够匹配另外一个内存条在3200频率点的SPD参数。"

> "这种频率匹配的算法有可能是这样的：首先比较两个内存在3200频率下的SPD数据，如果两组数据不同，选择其中数据较低（较容易达到）的那一组。"

> "以上只是诸多可能性中的【一种】。也就是两个内存条的SPD信息不一致（使用CPU-Z可以查看两个内存的SPD信息）。"

> "dnf install i2c-tools -y; modprobe eeprom; modprobe at24; modprobe i2c-i801; modprobe i2c-amd-mp2-pci; modprobe ee1004; decode-dimms"
