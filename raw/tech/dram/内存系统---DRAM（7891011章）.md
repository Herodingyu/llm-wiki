## 7.1 DRAM基础：内部，操作

DRAM使用一个晶体管+电容的组合来存储1-bit，具体结构如下图7.3，其电容必须定期刷新。

其可以以不同的合并/独立的方式组织，通常x8 DRAM表示的是有8个内存阵列同时工作，列位宽为8，每次可以读取8-bit数据。下图7.4展示了x2/x4/x8 DRAM的组织形式，其每个都有多个array，但是同属于一个bank。每个bank都是独立的，可以独立的激活/预充电/读数据。

一个系统可以有多个DIMM，每个可认为是独立的Bank，每个DIMM上的DRAM chip内部也可以实现多个独立的Bank，所以引入了Rank来区分这两个层级。具体下图7.5展示了当代DRAM系统的不同组织形式，层次从高到低分别是DIMM、Rank、Bank和Array。

Rank和Bank级提供的并发性可以提供高带宽服务。JEDEC风格的组织方式将总线划分为data, address, control, 和 chip-select，如下图7.6所示，其中chip select用于控制Rank级的并行（可以认为是地址的一部分）。

下图7.7展示了典型DRAM读写请求的顺序，MC充当了一个联络员，其与DRAM的接口相对复杂：

- row-address strobe (RAS)和column-address strobe (CAS)细节如下图7.8，分别对应row地址激活一个page和col地址激活部分bit所需的时间
- 在读取/写入数据时，需要先将差分放大器precharge到0/1电平的中间值，然后对应bit的电容释放即可感知其具体只。
- row/col地址、bank/rand id都需要通过地址译码得到（row/col地址分别传输，因为DRAM制造pin成本更高）

## 7.2 DRAM架构演进

下图7.10展示了DRAM架构从时钟驱动--->传统异步--->fast page mode (FPM)--->extended dataout (EDO)--->burst-mode EDO (BEDO)--->synchronous (SDRAM)的过程。SDRAM之后，演进分成了两个方向：低延时和高带宽

### 7.2.1 针对吞吐量的结构修整

1. FPM允许在多个CAS命令间保持打开状态，只需要微小的电路修改。
2. EDO则将输出驱动改为了锁存器输出，以保存有效数据更长时间。
3. BEDO增加了内部counter来驱动地址锁存，那么如果所需的地址和前几笔没有变化，则MC就不需要每次CAS都提供一个新的地址了
4. SDRAM更进一步通过时钟驱动内部电路

**传统异步DRAM**

每次DRAM访问都需要经过完整的步骤，即使是请求相同的数据，其时序图如下图7.11。

**FPM DRAM**

其在两次CAS命令之间可以保持打开状态，可以省掉一个RAS过程，时序图如下图7.12。

**EDO DRAM**

在感应放大器和输出引脚间增加了锁存器，以允许阵列可以更快的预充电，并且输出数据可以保持更长的时间。下图7.13给出了其时序图，其节省了数据传输的时间。

**BEDO DRAM**

BEDO在EDO的基础上增加了burst概念，从而不需要每次都完整的将地址从外部输入。下图7.14对应其时序图。

**Synchronous DRAM (SDRAM)**

上边的结构都是异步控制，而使用同的好处是可以让数据和控制信号以更加周期性的形式工作，可以具有更小的skew，而更低的skew则意味着turnaround更短，吞吐量更高。对应的时序图如下图7.15所示。

### 7.2.2 针对吞吐量的接口修改

**Rambus DRAM (RDRAM, Concurrent RDRAM, and Direct RDRAM)**

RDRAM与传统DRAM结构不同，其总线要窄的多，并且不使用专用的地址/控制/数据控制接口，而是共用一组bus。传输过程中，首先传输地址/控制包，然后传输数据包（这种设计导致其并发度受限）。下图7.16展示了一个典型的读事务。

为此，JEDEC风格的DRAM使用了单独的地址、命令和数据包，这样可以并行执行多个操作，具体例子如下图7.17。

上述并发设计的限制之一是地址位宽太窄，导致有时需要使用数据bus来传输地址，为此Rambus提出了“Direct RDRAM”，其增加了数据/地址bus的位宽，具体其时序图如下图7.18。

**Double Data Rate DRAM (DDR SDRAM)**

DDR通过同时在上升沿/下降沿传输数据来使得带宽翻倍，下图7.19展示了其具体操作。

### 7.2.3 针对延时的结构修改

**Virtual Channel Memory (VCDRAM)**

增加了SRAM Cache来存放更大的数据，且该SRAM可以显式的被MC控制。

**Enhanced SDRAM (ESDRAM)**

在column mux之前增加了latch，则其大小与DRAM page相同，可以更好的重叠activity。

**MoSys 1T-SRAM**

其将一个SRAM接口封装在一个DRAM core外

**Reduced Latency DRAM (RLDRAM)**

其直接连接到mc，而不是像传统DIMM一样是一个单独的pcb板卡。

**Fast Cycle DRAM (FCRAM)**

通过将大数组拆分成子数据来实现低延时数据访问，row激活时只有一个子数据被驱动。这样可以减少数据大小，减少被占用时间，进而降低latency。

## 7.3 现代DRAM标准

### 7.3.1 JEDEC SDRAM技术的显著特点

JEDEC SDRAM使用传统的DRAM组织形式，包含独立的data/address/control/chip-sel总线：

- data bus相对较宽
- address bus随着DRAM设备存储容量增多而增加
- control bus包含row/col strobe等信号
- chip-sel以rank为粒度

在DDR SDRAM中的写请求中，DRAM并不是根据时钟来采样输入数据，而是使用DQS信号。

**Single Data Rate SDRAM**

使用单沿进行数据传输和采样，同时增强了两项功能。

***Programmable Burst Length***

和BEDO DRAMs类似，其使用burst data来提高带宽。同时因为具有时钟信号，所以不再需要连续的CAS信号而是可以通过寄存器配置器burst length。

***Programmable CAS Latency***

CAS latency指的是SDRAM接收到CAS命令后返回数据所需的时钟周期数。将其设为不同值可以允许在不同工艺生产的不同代产品具有相同表现。

**Double Data Rate SDRAM**

DDR有几个新增feature：

- dual-edged clocking
- 单时钟沿驱动地址和控制信号，双时钟沿驱动数据信号，时序图如下图7.20
	- 需要一些额外的时钟补偿机制，比如制定了一个与时钟中心对齐的data strobe信号
- on-chip delay-locked loop (DLL)
- DLL用于延时现有周期信号使其与另一个信号同步
	- PLL使用振荡器产生与另一个信号同步的新周期信号
	- 当两者被用在通信接口中时，结果就是一个闭环电路，可以测量和消除发送/接收电路中的不确定性
- 片上DLL将DRAM输出和DQS与时钟信号做同步（针对读信号，写信号由MC对数据和DQS做同步）
	- MC通常有两个时钟，一个原始时钟，一个delay 90°后的时钟（使用delay后的时钟进行采样）
	- PLL和DLL的功能都是同步两个周期信号，使他们之间存在一定的相位差或者延时，两个属于经常交换使用
	- 下图7.21展示了在DDR SDRAM中使用的DLL及其影响

### 7.3.2 其他技术，特别是Rambus

**Rambus’ ’898 Patent Application**

其最突出的是物理总线的组织形式，下图7.22展示了其差异，其只使用了一个bus来传递所有信息。

另外一大特征是其总线位宽非常窄，并且使用不同的包格式来进行传输，所以更像计算机网络的包协议而不是传统存储接口。这就要求DRAM需要具备更多功能，比如解析packet，识别chip-sel结果，对dram的要求更高而不是像传统的仅仅作为一个slave。

***使用可变延时电路降低时钟skew***

Rambus的时钟方案用于与DRAM设备内不存在的内部时钟同步，结构如下图7.23，MC发出的全局时钟会被反射回来，而每个DRAM都有两个时钟输入。

***Variable Request Latency***

该机制允许MC指定DRAM在处理器请求时应该等待多长时间。

**Variable Block Size**

该机制允许MC指定需要读取或写入多少数据。

**Running the Clock at the Data Rate**

如下图7.25，下边的图示为降速后的时钟，这使得时钟更容易被处理，但是在MC内意味着一些数据没有对应的上升沿，所以需要一些额外操作进行恢复。

### 7.3.3 Rambus和JEDEC DRAM的比较

- Programmable CAS Latency
- JEDEC该特性的目的是让不同vendor各自优化器性能，所以只在初始化的时候设置
	- Rambus可以每次发送请求时设置
- Programmable Burst Length
- 同样是在初始化时设置
	- Rambus中的可变块大小更小，同时可以运行时设置该参数
- Dual-Edged Clocking
- 两者使用该特性
	- JEDEC的DRAM中内部定时信号是由时钟和DQS信号一起产生的
	- Rambus风格中，定时信号没有时钟源，并使用U型全局信号的不同相位来产生的
- On-Chip PLL/DLL
- JEDEC中使用片上DLL来确保数据和全局时钟同步
	- Rambus则使用片上可变延时电路来实现

## 7.4 全缓冲DIMM：一种折中

DRAM行业最近的一个有趣发展是FB-DIMM，类似于传统DRAM和Rambus DRAM间的折中。随着DRAM速度变快，其所允许的channel数量也越来越少，DDR3就只能容纳一个channel，这限制了内存容量。

FB-DIMM的结构如下图7.26，这样可以使得在获得和图形卡类似的性能同时取得和传统DRAM类似的容量。其通过将MC移动到DIMM使得所有的连接都变成了点对点的形式

## 7.5 DRAM系统的问题

- 架构和演进
- 每一代DRAM的带宽提升都是以channel数量和访问粒度为代价的。通过以空间换时间的策略，可以在不提高DRAM core速度的情况下提供DRAM IO的速率。
	- DDR相比SDR提升为2n预取：从DRAM core中预取两倍bit，DDR2提升到4n预取，DDR3提升到8n预取。
- 拓扑和时序
- 信号完整性与系统/拓扑组织形式有着很大关系。
- 引脚和协议效率
- 功耗和散热

## 8.4 差分感应放大器

### 8.4.1 DRAM设备中感应放大器的作用

其组要有三个功能：

1. 当接入晶体管时，电容器中的电荷会引起bitline上微小的电压变化，然后将其与另一条做比较和可以解析出数据0/1
2. 在bitline上的电压被检测和放大后还可以恢复cell内的值
3. 可以充当临时的数据存储元件，在下一层预充电之前都可以保持数据用于下一次访问

### 8.4.2 基本感应放大器的电路设计

其基本电路结构如下图8.10，其中

- EQ信号控制电压均衡电路，可用于预充电，保证bitline对上的电压尽可能匹配。
- 另一侧的感应电炉，在SAN和SAP被驱动时，会将bitline驱动到完整的电压水平。
- 然后CSL打开输出晶体管将full电压输出
- 与此同时，被读取的cell仍保持打开状态，bitline又反向将数据写回cell
- 另外，如果是写操作，CSL和WE共同作用下可以允许外部输入来驱动感应放大器和bitline的电压

### 8.4.3 基本的感应放大器操作

下图8.11展示了感应放大器在操作中的4种不同相位：

- precharge
- 在开始读取数据之前，需要将DRAM阵列中的bitline预充电到参考电压
- access
- 选中某一个cell时，对应的bitline受到电容的影响会进行一定扰动
- sense
- 感应电炉开始工作，放大bitline上的电压差
- restore
- 在bitline被驱动到full电压之后，晶体管保持开启，电容会再次被回复原始状态
	- 同时存储值被读出

下图9.29说明了高速数字电路中的一个问题，即时钟传播延时过大，这是就需要重新进行同步。

- 全局时钟系统
- 认为信号在系统中不同设备间的传播延时和时钟周期长度相比是比较小的

- RDRAM和DDR SDRAM时钟该方案

Row也被称为DRAM page，因为行激活命令会在感应放大器上缓存一个page，直到下一次预充电命令。

## 11.3 额外约束

上述的时序参数用于指定不同DRAM命令间组合的最小时序约束，但现代DRAM存储系统中还存在着额外约束回限制其带宽利用率。具体与DRAM的功耗及内存系统向DRAM设备分配的命令和地址有关。

### 11.3.1 设备功耗限制

DRAM制造商推动着每一代DRAM设备的速率不断增加，同时也导致了更高的功耗。一种降低功耗的策略是降低其activity rate，其反过来也会限制DRAM的数据移动能力，进而限制系统性能。

行激活是一个相对耗能的操作，因为需要对数千bit并行的discharge、sensed和restore。下图11.30展示了一个读操作过程中的电流profile。

下图11.31展示了连续两个读操作的电流profile。

### 11.3.2 tRRD: Row-to-Row (Activation) Delay

### 11.3.3 tFAW: Four-Bank Activation Window

随着DRAM容量越来越大，如何确定其bank count/row count/row size呢？

- 因为Bank控制逻辑的开销，所以制造商不愿意增加bank数
- 更大的row则会导致一个行激活命令造成更大的功耗
- 增加row数量则意味着每个bitline对应更多bit，会造成更长的和更多的row刷新命令

为了限制大容量DDR2设备中功耗的增加，定义了时序参数标识一个滚动time-frame，同一DRAM设备上最多可以同时有4 row激活。下图11.32展示了这样受限的访问时序。
