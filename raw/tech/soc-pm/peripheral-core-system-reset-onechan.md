# 三类复位：外设复位、核复位与系统级复位的本质差异

> 来源：微信公众号「OneChan」
> 原文链接：https://mp.weixin.qq.com/s/GNdptYjZeBQ6YSzR-_AVNw
> 记录时间：2026-05-04

---

## 核心观点

复位的本质是：**硬件复位控制器向目标模块下发异步/同步清零信号，强制模块寄存器、状态机、流水线、缓存、锁存器回到芯片出厂默认上电初始态**。

三类复位的核心差异在于：**作用范围、电源/时钟关联性、寄存器影响域、流水线与上下文保留、复位触发源、故障隔离能力**。

---

## 三类复位底层定义

### 1. 外设复位（Peripheral Reset）

**本质**：最小粒度的局部模块复位，仅针对独立外设 IP 的本地状态机、控制寄存器、FIFO、中断锁存、硬件计数器做强制初始化。

**不触碰**：内核、总线矩阵、系统全局配置、内存、时钟根节点。

| 维度 | 表现 |
|------|------|
| 复位域 | 单一外设独立复位寄存器位控制（如 RCC_APB1RST、RCC_AHB2RST） |
| 时钟/电源 | 内核时钟、系统总线时钟持续运行，全局电源域不掉电 |
| 隔离性 | 完全局部隔离，复位期间 CPU 正常跑代码、其他外设正常工作 |

**典型场景**：串口卡死、定时器计数溢出锁死、DMA 传输异常。

**底层表现**：
- 总线不中断，CPU 无需暂停
- 外设内部状态机强制跳转空闲态，错误标志位硬件自动清除
- 不影响其他外设通信、DMA 搬运、定时中断

### 2. 核复位（Core Reset / CPU Core Reset）

**本质**：CPU 内核域专属复位，只复位处理器核心内部硬件资源，**保留片上总线、全局外设、SRAM/Flash、系统控制寄存器、时钟树、电源配置**。

| 维度 | 表现 |
|------|------|
| 复位域 | CPU 流水线、通用寄存器(R0~R15)、PSR 状态寄存器、栈指针、MPU、FPU、NVIC、核心缓存 |
| 时钟/电源 | 系统全局时钟、总线时钟、外设时钟保持锁存，电源域稳定 |
| 隔离性 | 内核独立复位，外设、DMA、总线矩阵、片上存储全程不复位 |

**典型场景**：程序跑飞、数组越界引发内核 Fault、任务栈溢出。

**底层表现**：
- 总线矩阵、DMA、GPIO 配置、外部硬件状态**完全保持**
- 仅内核流水线冲刷、寄存器清零
- **快速软重启**，复位耗时最短

### 3. 系统级复位（System Reset）

**本质**：全域半完整复位，覆盖**内核 + 总线 + 全局外设 + 系统控制层**，是除备份电源域（BKP）、独立低速时钟（LSE）以外的大面积系统初始化。

| 维度 | 表现 |
|------|------|
| 复位域 | CPU 内核 + 全部片内外设 + 总线矩阵 + DMA + 时钟配置寄存器 + GPIO 全局配置 + 低功耗控制模块 |
| 时钟/电源 | 高速时钟（HSE/HSI）重新初始化，时钟树分频/倍频配置清零，主电源域保持，备份域独立保留 |
| 隔离性 | 无局部隔离，全域业务模块统一回归默认态 |

**典型场景**：时钟配置错乱、多外设资源冲突、低功耗模式卡死、全局配置错误。

**底层表现**：
- 所有外设强制回归默认，复用功能、引脚配置全部清空
- 时钟 PLL、分频系数关闭重置
- 仅备份域（RTC、备份寄存器）保留

## 关键维度对比

| 对比维度 | 外设复位 | 核复位 | 系统级复位 |
|---------|---------|--------|-----------|
| 作用边界 | 单个外设 IP | 仅 CPU 核心内部 | 内核 + 全外设 + 总线 + 系统配置 |
| 寄存器影响 | 外设 CR/SR/FIFO/中断挂起位清零 | 内核通用寄存器、NVIC、流水线清零 | 全局外设寄存器 + 内核 + 时钟配置全部清零 |
| 代码上下文 | CPU 程序正常执行，无 PC/SP 重置 | PC、SP、寄存器全部重置，程序从头执行 | 程序重头启动，全局变量 RAM 随机化 |
| 时钟树状态 | 全局时钟不变 | 全局时钟、分频倍频完全保留 | 高速时钟重置，时钟配置恢复出厂默认 |
| SRAM/Flash | 存储数据完整保留 | 全局 SRAM、Flash 数据完整保留 | 普通 SRAM 数据丢失，Flash 程序区只读保留 |
| 故障范围 | 仅修复单外设卡死、FIFO 溢出、状态机死锁 | 修复内核死循环、异常指令、总线 Fault | 解决全局总线异常、多外设冲突、时钟配置错误 |
| 触发方式 | 软件单独置位复位位、外设硬件错误触发 | 软件 SYSRESETREQ、看门狗短复位、内核 HardFault | 软件系统复位、独立看门狗、NRST 引脚、严重系统故障 |

## 与上电复位（POR）的边界区分

| 特性 | 三类复位 | 上电复位（POR） |
|------|---------|----------------|
| 电源状态 | 热复位，主电源不掉电 | 冷启动，电源从无到有 |
| 备份域 | 保留 | 复位 |
| 调试模块 | 保留 | 复位 |
| 芯片熔丝配置 | 保留 | 复位 |
| PMIC 初始化 | 无需 | 必须 |

**系统级复位 ≠ 上电复位**：上电复位会复位备份电源域、调试模块、芯片熔丝配置，是芯片最彻底的复位。

---

## 工程避坑口诀

- **外设故障优先用「外设复位」**，避免过度复位拖慢系统实时性
- **仅内核异常用「核复位」**，保留硬件外设状态，减少外设重初始化代码
- **全局架构异常、时钟错乱才使用「系统级复位」**，防止频繁全域复位导致设备断连、业务中断

## 附录：Cortex-M 寄存器实操代码

### 关键核心寄存器定义
```c
// SCB 系统控制块
#define SCB_BASE        0xE000ED00U
#define SCB_AIRCR       (*((volatile uint32_t *)(SCB_BASE + 0x0CU)))

#define AIRCR_VECTKEY      (0x5FAUL << 16)   // 密钥：必须写入，否则复位无效
#define AIRCR_SYSRESETREQ  (1UL << 2)        // 核复位 / 系统复位触发位
#define AIRCR_VECTCLRACTIVE (1UL << 1)       // 中断向量清除位

// RCC 外设复位寄存器
#define RCC_APB1RSTR    (*((volatile uint32_t*)0x40021010U))
#define RCC_APB1_UART2RST  (1UL << 17)
```

### 外设复位
```c
void Peripheral_Reset(volatile uint32_t *RccRstReg, uint32_t RstBit) {
    *RccRstReg |= RstBit;     // 置位：下发外设硬件复位信号
    __NOP(); __NOP(); __NOP(); // 短延时：等待状态机完成清零
    *RccRstReg &= ~RstBit;    // 清零：解除复位
}
// 调用：Peripheral_Reset(&RCC_APB1RSTR, RCC_APB1_UART2RST);
```

### 核复位
```c
void Core_Only_Reset(void) {
    __disable_irq();
    SCB_AIRCR = AIRCR_VECTKEY | AIRCR_VECTCLRACTIVE;  // 只触发核心域复位
    while(1) { __NOP(); }
}
```

### 系统级复位
```c
void System_Global_Reset(void) {
    __disable_irq();
    SCB_AIRCR = AIRCR_VECTKEY | AIRCR_SYSRESETREQ;  // 触发芯片系统复位链路
    while(1) { __NOP(); }
}
```

### HardFault 自动触发核复位
```c
void HardFault_Handler(void) {
    Core_Only_Reset();  // 发生硬件异常，直接执行纯核复位快速恢复
}
```

## 关键底层差异 & 避坑要点

| 要点 | 说明 |
|------|------|
| **密钥必写** | AIRCR 无 0x5FA 密钥，复位位写入无效，属于 Cortex-M 内核硬件安全机制 |
| **复位延时原则** | 外设复位必须加 NOP 延时，高速总线下极易出现「复位未完成就写寄存器」导致硬件锁死 |
| **复位选型口诀** | 外设卡死用局部，内核跑飞用核复，时钟乱、全局异常用系统复位 |
| **RAM 数据差异** | 外设/核复位：SRAM 数据完全保留；系统复位：动态 RAM 数据随机丢失 |
