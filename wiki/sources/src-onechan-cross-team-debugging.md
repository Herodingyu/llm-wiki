# 跨团队协同调试：破解"这不是我的问题"死循环

## 核心主题

跨团队调试的核心瓶颈不是技术问题，而是责任边界模糊导致的心理防御机制，需通过系统化方法论和沟通艺术破解。

## 关键内容

- 问题本质：硬件/软件/验证团队间的"这不是我的问题"死循环
- 五步方法论：建立问题共有文化→创建单一事实源→最小复现原则→证据链闭环→共同制定验证方案
- 沟通黄金法则：先共情后讲理、用数据说话、留面子给台阶、及时同步、庆祝解决
- 禁止话术清单：5种破坏协作的典型表达方式
- 实战模板：问题报告模板、联合调试会议模板、根因确认模板
- 单一事实源格式：时间戳/团队/现象/已排查项/下一步的标准化记录

## 技术亮点

| 亮点 | 说明 |
|------|------|
| 五步方法论 | 从文化建立到根因确认的完整流程，每一步都有具体行动项 |
| 单一事实源 | 跨团队共享调试日志的标准格式，消除信息孤岛 |
| 最小复现原则 | "如果10行代码能复现，就不要用1000行"的实战准则 |
| 证据链闭环 | "示波器显示时钟丢失"替代"我觉得是时钟问题"的数据驱动文化 |
| 实战模板 | 可直接复制使用的三大调试文档模板 |
