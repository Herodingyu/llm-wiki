# SoC常用外设接口  串行协议IIC、UART、SPI对比

## 来源

- **原始文件**: raw/tech/peripheral/SoC常用外设接口--串行协议IIC、UART、SPI对比.md
- **提取日期**: 2026-05-03

## Summary

本文对SoC（System on Chip）系统中三种最常用的串行通信协议——IIC（I2C）、UART和SPI进行了全面的横向对比分析。在嵌入式系统设计中，选择合适的通信协议对系统性能、成本和可靠性有重大影响。文章从信号线数量、通信方式、速度、拓扑结构、寻址机制、流控制、功耗和典型应用场景等多个维度对三种协议进行了详细比较。I2C以其极简的两线设计和多设备管理能力适合传感器和低速外设；SPI以全双工高速传输和简单的硬件接口适合存储器和高速数据流；UART以异步通信的灵活性和点对点全双工能力适合调试和模块间通信。理解三种协议的特点和适用场景，是SoC系统架构师和嵌入式工程师进行外设接口选型的基础。

## Key Points

### 三种串行协议全面对比

| 特性 | I2C | SPI | UART |
|------|-----|-----|------|
| 信号线 | 2（SDA, SCL） | 4+（SCLK, MOSI, MISO, CS） | 2（TX, RX） |
| 通信方式 | 同步半双工 | 同步全双工 | 异步全双工 |
| 时钟 | 共享SCL | 共享SCLK | 无（双方约定波特率） |
| 速度 | 100K-3.4Mbps | 几Mbps-几十Mbps | 9600-数Mbps |
| 拓扑 | 总线（多主多从） | 星形（一主多从） | 点对点 |
| 寻址 | 7/10位设备地址 | 片选信号（CS） | 无（点对点直接连接） |
| 流控制 | 时钟拉伸 | 无（需额外处理） | RTS/CTS或XON/XOFF |
| 连接数 | 最多127（7位） | 受限于CS引脚数 | 1对1 |
| 功耗 | 低（开漏） | 中等 | 低 |
| 硬件复杂度 | 简单 | 简单 | 最简单 |
| 软件复杂度 | 中等 | 简单 | 简单 |

### I2C适用场景

1. **传感器网络**
   - 温度、湿度、压力传感器
   - 加速度计、陀螺仪
   - 环境光、接近传感器

2. **存储器**
   - EEPROM（24C系列）
   - 小型配置存储

3. **显示和触摸**
   - LCD控制器
   - 电容/电阻触摸屏

4. **电源管理**
   - PMIC
   - 电池管理芯片

### SPI适用场景

1. **高速存储**
   - SPI Flash（Nor/Nand）
   - SD卡（SPI模式）

2. **显示接口**
   - TFT LCD（高刷新率）
   - OLED显示屏

3. **通信模块**
   - Wi-Fi模块（ESP8266等）
   - 部分LoRa模块

4. **数据采集**
   - ADC（高速采样）
   - 部分DAC

### UART适用场景

1. **调试接口**
   - 串口控制台（console）
   - 固件下载和升级

2. **模块通信**
   - GPS模块
   - 蓝牙模块（HC-05等）
   - GSM/GPRS模块

3. **工业通信**
   - RS-232设备
   - RS-485总线（需转换）

4. **低速数据**
   - 传感器简单数据上报
   - 配置命令传输

### 选型决策树

```
需要多设备共享总线?
├── 是 → 需要地址寻址?
│   ├── 是 → I2C（传感器、EEPROM）
│   └── 否 → 考虑RS-485/UART扩展
└── 否 → 需要高速传输?
    ├── 是 → 需要全双工?
    │   ├── 是 → SPI（Flash、高速ADC）
    │   └── 否 → 考虑I2C高速模式
    └── 否 → 需要异步/长距离?
        ├── 是 → UART/RS-232/RS-485
        └── 否 → SPI或I2C均可
```
