## Summary

本文介绍了可信计算基础（TCB, Trusted Computing Base）的核心概念，澄清了"trusted"在计算机安全语境中的特殊含义——指对系统安全至关重要的组件，而非日常语义中的"安全"或"可信任"。文章阐述了TCB的组成要素、与TPM等特定硬件的区别，以及验证TCB可信度的四种方法：形式验证、加密测量（如Red Hat的Keylime项目）、可审计性（开源代码优势）和篡改证据（硬件信任根如TPM）。同时梳理了可信计算领域的相关术语：TC（可信计算）、TPM（可信平台模块）、TCM（可信密码模块）、TPCM（可信平台控制模块）、TSS（可信软件栈）、TNC（可信网络连接）。

## Key Points

### 1. TCB 定义与核心概念
- **定义**：对建立和维护特定系统安全性至关重要的所有系统组件的集合
- **与TPM区别**：TCB不是特定芯片或规范，而是系统安全体系结构中的术语
- **组件差异**：不同系统的TCB包含的组件可能不同，需具体分析

### 2. Trusted ≠ Secure
- **日常语义**：可信 = 可靠、真实、值得信任
- **安全语境**：可信 = 对系统范围内的安全至关重要
- **关键澄清**：可信组件不意味着能抵御攻击，只是其失效会导致系统受损
- **审查要求**：所有可信组件都需要额外审查以确保其值得信任

### 3. 验证 TCB 可信度的方法
| 方法 | 说明 | 示例 |
|------|------|------|
| **形式验证** | 数学证明系统行为符合预期 | 高安全等级系统 |
| **加密测量** | 可外部检查或审计的证明 | Red Hat Keylime项目 |
| **可审计性** | 开源代码供广泛社区检查 | Linux开源生态 |
| **篡改证据** | 检测是否被篡改，难度随堆栈高度增加 | TPM硬件信任根 |

### 4. 相关术语速查
- **TC**：可信计算（Trusted Computing）
- **TPM**：可信平台模块，提供可信根的芯片
- **TCM**：可信密码模块，国内标准，与TPM对应
- **TPCM**：可信平台控制模块，具有平台资源控制功能
- **TSS**：可信软件栈，为OS和应用提供TPM接口
- **TNC**：可信网络连接，实现平台到网络的可信扩展

## Key Quotes

> "可信计算库（TCB）并不像可信平台模块（TPM）那样指代特定的芯片或规范。系统的可信计算基础TCB是安全体系结构中的一个术语，指的是对建立和维护特定系统的安全性至关重要的所有系统组件。"

> "在计算机安全的上下文中，'可信'只是指对系统范围内的安全至关重要。请注意，这个定义并没有说明受信任组件是否能够抵御任何攻击，只是说它在系统的安全性中发挥着不可或缺的作用。"

> "可信度尽可能植根于形式验证或加密测量，这可以从数学上证明系统或其组件的行为符合预期。"

> "出于这个原因，许多旨在更安全的系统都包含了硬件信任根，例如可信平台模块（TPM）。"
