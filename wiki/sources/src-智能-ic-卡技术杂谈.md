## Summary

本文是知乎专栏「成都IT圈」发布的关于智能IC卡技术的 comprehensive 综述文章，系统介绍了集成电路卡（IC卡）的技术架构、通信协议、安全机制以及主流产品生态。IC卡作为一种内嵌微控制器、存储器和加密算法协处理器的塑料卡片，已广泛应用于金融支付、身份认证、公共交通、门禁控制等领域。文章从IC卡的基本分类（接触式、非接触式、双界面卡）出发，详细解析了ISO/IEC 7816接触式卡协议、ISO/IEC 14443非接触式卡协议（Type A/B）、恩智浦Mifare Classic系列卡片的工作原理与安全特性，并深入探讨了应用协议数据单元（APDU）格式、安全元件（SE）、卡片操作系统（COS）以及ARM SecurCore系列安全处理器等核心技术。文章内容丰富、图文并茂，为从事智能卡开发、安全芯片设计以及物联网安全应用的工程师提供了宝贵的技术参考。

## Key Points

### IC卡分类与形态

| 类型 | 通信方式 | 工作频率 | 典型应用 |
|------|---------|---------|---------|
| 接触式卡 | 物理触点连接 | - | 银行卡、SIM卡、身份证 |
| 非接触式卡 | 射频感应（RFID） | 13.56MHz | 公交卡、门禁卡、NFC标签 |
| 双界面卡 | 接触+非接触两用 | 13.56MHz | 高端金融卡、多功能市民卡 |

### ISO/IEC 7816 接触式卡协议

该标准由14个部分组成，核心内容包括：

- **ISO7816-1**：物理特性（尺寸、材质、温度范围）
- **ISO7816-2**：触点尺寸与位置（C1-C8定义）
- **ISO7816-3**：电子接口与传输协议（T=0/T=1）
- **ISO7816-4**：APDU命令结构、文件组织、安全体系
- **ISO7816-8**：安全操作命令与加密机制

> 注：不存在ISO 7816-14标准。

### ISO/IEC 14443 非接触式卡协议

| 特性 | Type A | Type B |
|------|--------|--------|
| 提出厂商 | NXP等 | ST等 |
| 调制方式 | 100% ASK | 10% ASK |
| 编码方案 | 改进米勒编码 | NRZ编码 |
| 抗干扰性 | 信息区别明显 | 持续能量传递 |
| 能量稳定性 | 高电压时可能波动 | 更稳定 |

**工作原理**：读写卡设备（PCD）向IC卡（PICC）发送13.56MHz电磁波，包含电源信号和指令数据信号两部分：

1. **电源信号**：IC卡内置LC谐振电路接收电磁波并整流储能，达到2V即可供电
2. **指令信号**：PCD指挥IC芯片完成数据读取、修改、存储等操作

### Mifare Classic（M1卡）技术细节

恩智浦1994年推出的Mifare Classic系列，工作频率13.56MHz，符合ISO 14443 Type A规范：

**存储结构**：
- 16个扇区（Sectors）
- 每个扇区4个数据块（Blocks），每块16字节
- 第0扇区块0为厂商信息（32位序列号，只读）
- 每扇区块3为扇区尾部（密码A + 存取控制 + 密码B）

**卡片类型**：

| 类型 | 可修改性 | 特性 |
|------|---------|------|
| 标准M1卡 | 第0扇区不可改 | 最常用，兼容性好 |
| UID卡 | 全区块可改写 | 响应后门指令，可被检测 |
| CUID卡 | 全区块可改写 | 不响应后门指令，防检测 |
| FUID卡 | 0区块仅写一次 | 可绕开部分反克隆设备 |

**访问流程**：
1. 复位应答（ATQA响应）
2. 防碰撞循环（选择单张卡片）
3. 选择IC卡（SAK响应）
4. 三次互相确认（密码校验）
5. 执行读写/加减/恢复/传输/停止操作

### 应用协议数据单元（APDU）

APDU是卡片与外部应用之间的通信报文协议，定义在ISO7816-4中：

- **命令APDU**：由读写设备发送到智能卡
  - 必选4字节头部：`CLA + INS + P1 + P2`
  - 可选数据字段：0~255字节

- **响应APDU**：由智能卡返回给读写设备
  - 必选2字节状态字
  - 可选数据字段：0~255字节

### 安全元件（SE）与卡片操作系统（COS）

**安全元件（SE）**：
- 由安全硬件和软件组成
- 提供安全运行环境、存储、算法、接口
- 实现身份认证、数据加密、敏感信息保护
- 物联网安全四大需求：设备唯一标识、双向认证、加密传输、安全OTA

**卡片操作系统（COS）**：
- 紧密围绕智能卡特点设计
- 遵循ISO/IEC 7816或ISO/IEC 14443标准
- 控制信息交换、管理存储器、处理内部命令
- 本质更接近监控程序而非完整操作系统
- **Java Card**已成为COS事实上的工业标准

### 主流IC卡芯片厂商

| 厂商 | 系列 | 特点 |
|------|------|------|
| NXP | Mifare系列 | 市场占有率高，Classic算法已被破解 |
| Infineon | SL系列 | 高安全性，金融领域应用广泛 |

## Key Quotes

> "集成电路卡又称为IC卡（Integrated Circuit Card），是一种内嵌有集成电路的塑料标签或卡片，其集成电路当中包含有8位或者32位的微控制单元MCU、只读存储器ROM、电可擦只读存储器EEPROM或者闪速存储器Flash、随机访问存储器RAM，以及固化在只读存储器ROM当中的片内操作系统（COS，Chip Operating System）。"

> "ISO/IEC 7816是一种标准化的接触式智能卡通信协议，主要用于读写接触式的集成电路卡，该协议由如下14个部分组成。"

> "经历了三级两层映射：INTx -> PIRQx -> IRQx。这个映射关系是中断路由的精髓。"

> "应用协议数据单元（APDU，Application Protocol Data Unit）是卡片和外部应用之间的通信报文协议，其格式标准被定义在ISO7816-4当中。"

> "伴随近年5G逐步组网商用，物联网在迎来较快发展的同时，安全问题日益突出，基于嵌入式安全元件SE提供的安全存储与运算环境，就可以为物联网设备运营者提供一个安全可信任的根。"
