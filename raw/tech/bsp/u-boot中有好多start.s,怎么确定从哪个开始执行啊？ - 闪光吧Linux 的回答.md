一、开篇 在u-boot中， struct driver 用于描述某个功能或外设的驱动程序，该结构体保存了用于初始化（setup）和移除（remove）设备的方法。设备在初始化自身时需要相关信息： 这些信息要么来自于 platdata ，要么来自于设备树节点（device tree node） ，驱动程序通过匹配 compatible 字符串（ of\_match ）找到对应的节点。 每个驱动程序都属于某个 uclass ，该 表示一类同类型设备。驱动程序中通用的部分可以在 中实现，同时 还可以为其下所有驱动提供统一的接口。 实现如下（/include/dm/device.h）： struct driver { /\* 驱动名称 \*/ char \*name;
