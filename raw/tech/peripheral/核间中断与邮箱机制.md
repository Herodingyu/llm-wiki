## 第10章 多核通信与同步机制

## 10.1 核间中断与邮箱机制

### 10.1.1 设计来源与设计思想

**设计来源：**  
核间中断与邮箱机制的设计源于多核处理器架构对高效通信和同步的迫切需求，其技术思想融合了多个领域的先进理念：

1. **多核处理器架构演进** ：从对称多处理（SMP）到非对称多处理（AMP）架构，需要灵活的核间通信机制
2. **实时系统确定性需求** ：汽车电子和工业控制要求确定性的通信延迟和可靠的同步
3. **分布式系统理论** ：借鉴消息传递接口（MPI）和远程过程调用（RPC）的通信模式
4. **ARM GICv3架构规范** ：ARM通用中断控制器v3为核间中断提供硬件级支持
5. **车规芯片功能安全要求** ：ISO 26262标准要求通信机制具备错误检测和容错能力

**设计思想演进：**  
从简单的共享内存到复杂的通信协议，体现了通信机制的成熟化过程：

![图片](https://mmbiz.qpic.cn/mmbiz_png/QTnjRdicb9qZpNjf4VcIT3CXQvoapliaTHibal2ZDjKFe3hAZ8M1SIKOaD05Du3vjhcQ7RltDg5STAelKSmoHwzZw/640?wx_fmt=png&from=appmsg&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=0)

**核心设计原则：**

1. **低延迟原则** ：通信机制必须最小化传输延迟，满足实时性要求
2. **确定性原则** ：通信时间必须可预测，支持最坏情况时间分析
3. **可靠性原则** ：消息传递必须可靠，支持错误检测和重传
4. **可扩展性原则** ：机制必须支持不同数量的核心和拓扑结构
5. **简单性原则** ：硬件和软件接口应简单易用，降低使用复杂度

#### 10.1.3.2 关键机制详解

**机制1：中断与邮箱的协同工作**  
中断提供快速通知，邮箱提供数据传递，两者协同实现高效通信：

![图片](data:image/svg+xml,%3C%3Fxml version='1.0' encoding='UTF-8'%3F%3E%3Csvg width='1px' height='1px' viewBox='0 0 1 1' version='1.1' xmlns='http://www.w3.org/2000/svg' xmlns:xlink='http://www.w3.org/1999/xlink'%3E%3Ctitle%3E%3C/title%3E%3Cg stroke='none' stroke-width='1' fill='none' fill-rule='evenodd' fill-opacity='0'%3E%3Cg transform='translate(-249.000000, -126.000000)' fill='%23FFFFFF'%3E%3Crect x='249' y='126' width='1' height='1'%3E%3C/rect%3E%3C/g%3E%3C/g%3E%3C/svg%3E)

**协同工作机制原理：**

- **中断作为信号量** ：中断通知接收方有数据到达，避免轮询开销
- **邮箱作为数据缓冲区** ：邮箱存储消息数据，支持大数据传输
- **分离控制与数据** ：控制信息通过中断传递，数据通过邮箱传递
- **支持异步通信** ：发送方无需等待接收方立即处理

**机制2：可靠消息传递协议**  
基于序列号和确认的可靠消息传递机制：

![图片](data:image/svg+xml,%3C%3Fxml version='1.0' encoding='UTF-8'%3F%3E%3Csvg width='1px' height='1px' viewBox='0 0 1 1' version='1.1' xmlns='http://www.w3.org/2000/svg' xmlns:xlink='http://www.w3.org/1999/xlink'%3E%3Ctitle%3E%3C/title%3E%3Cg stroke='none' stroke-width='1' fill='none' fill-rule='evenodd' fill-opacity='0'%3E%3Cg transform='translate(-249.000000, -126.000000)' fill='%23FFFFFF'%3E%3Crect x='249' y='126' width='1' height='1'%3E%3C/rect%3E%3C/g%3E%3C/g%3E%3C/svg%3E)

**可靠消息传递原理：**

- **序列号机制** ：每个消息有唯一序列号，检测丢失和重复
- **确认机制** ：接收方确认收到消息，发送方确保消息到达
- **超时重传** ：发送方超时未收到确认则重传消息
- **滑动窗口** ：支持多个未确认消息，提高吞吐量

 * 发送核间中断
 * 向指定核心发送中断请求
 */
ipi_result_tipi_send(uint32_t target_core,ipi_type_t type,
uint32_t data,ipi_priority_t priority){

uint32_t retry_count =0;

// 1. 参数检查
if(target_core >= MAX_CORES){
IPI_LOG("Invalid target core: %d", target_core);
return IPI_INVALID_TARGET;
}

if(!is_core_online(target_core)){
IPI_LOG("Target core %d is not online", target_core);
return IPI_TARGET_OFFLINE;
}

// 2. 准备中断命令
    command.target_core = target_core;
    command.type = type;
    command.data = data;
    command.priority = priority;
    command.sequence =get_next_sequence_number();
    command.timestamp =get_system_timestamp();

// 3. 检查目标核心的中断屏蔽状态
if(is_ipi_masked(target_core, type)){
IPI_LOG("IPI type %d is masked for core %d", type, target_core);

// 根据策略处理：等待或返回错误
if(get_ipi_policy().wait_if_masked){
uint32_t wait_time =0;
while(is_ipi_masked(target_core, type)){
if(wait_time++> MAX_MASK_WAIT_TIME){
IPI_LOG("Timeout waiting for IPI mask to clear");
return IPI_MASK_TIMEOUT;
}
delay_us(100);
}
}else{
return IPI_MASKED;
}
}

// 4. 发送中断（支持重试）
while(retry_count < MAX_IPI_RETRY_COUNT){
ipi_result_t result =send_ipi_command(&command);

if(result == IPI_OK){
IPI_LOG("IPI sent successfully to core %d, type %d, seq %d",
                   target_core, type, command.sequence);

// 5. 等待确认（如果需要）
if(type == IPI_TYPE_RELIABLE){
if(wait_for_ipi_ack(target_core, command.sequence,
                                    IPI_ACK_TIMEOUT)== IPI_OK){
IPI_LOG("IPI acknowledged by core %d", target_core);
return IPI_OK;
}else{
IPI_LOG("IPI ack timeout, retrying...");
                    retry_count++;
continue;
}
}else{
return IPI_OK;
}
}else{
IPI_LOG("Failed to send IPI, error: %d, retrying...", result);
            retry_count++;
delay_us(RETRY_DELAY_US *(1<< retry_count));// 指数退避
}
}

IPI_LOG("Failed to send IPI after %d retries", MAX_IPI_RETRY_COUNT);
return IPI_SEND_FAILED;
}

/**
 * 核间中断处理程序
 * 处理接收到的核间中断
 */
voidipi_handler(uint32_t core_id){
ipi_status_t status;
ipi_command_t command;

// 1. 读取中断状态
if(read_ipi_status(core_id,&status)!= IPI_OK){
IPI_LOG("Core %d: Failed to read IPI status", core_id);
return;
}

// 2. 根据中断类型处理
switch(status.type){
case IPI_TYPE_NORMAL:
// 普通中断，处理并返回
handle_normal_ipi(core_id, status.data);
break;

case IPI_TYPE_RELIABLE:
// 可靠中断，需要确认
if(read_ipi_command(core_id,&command)== IPI_OK){
// 发送确认
send_ipi_ack(core_id, command.sequence);

// 处理命令
handle_reliable_ipi(core_id,&command);
}
break;

case IPI_TYPE_BROADCAST:
// 广播中断，所有核心都会收到
handle_broadcast_ipi(core_id, status.data);
break;

case IPI_TYPE_URGENT:
// 紧急中断，最高优先级处理
handle_urgent_ipi(core_id, status.data);
break;

case IPI_TYPE_MAILBOX:
// 邮箱中断，检查邮箱状态
handle_mailbox_ipi(core_id);
break;

default:
IPI_LOG("Core %d: Unknown IPI type: %d", core_id, status.type);
break;
}

// 3. 清除中断状态
clear_ipi_status(core_id);

// 4. 记录中断处理完成
record_ipi_handled(core_id,&status);
}

 * 邮箱结构定义
 */
typedefstruct{
volatileuint32_t head;// 读指针
volatileuint32_t tail;// 写指针
volatileuint32_t size;// 缓冲区大小
volatileuint32_t flags;// 状态标志
uint8_t*buffer;// 数据缓冲区

/**
 * 初始化邮箱
 */
mailbox_result_tmailbox_init(mailbox_t*mailbox,uint32_t core_id,
uint32_t buffer_size){
// 参数检查
if(mailbox ==NULL){
MAILBOX_LOG("Invalid mailbox pointer");
return MAILBOX_INVALID_PARAM;
}

if(buffer_size ==0|| buffer_size > MAX_MAILBOX_SIZE){
MAILBOX_LOG("Invalid buffer size: %d", buffer_size);
return MAILBOX_INVALID_SIZE;
}

MAILBOX_LOG("Initializing mailbox for core %d, size %d",
               core_id, buffer_size);

// 1. 分配缓冲区内存
    mailbox->buffer =(uint8_t*)malloc(buffer_size);
if(mailbox->buffer ==NULL){
MAILBOX_LOG("Failed to allocate mailbox buffer");
return MAILBOX_MEMORY_ERROR;
}

// 2. 初始化指针和计数
    mailbox->head =0;
    mailbox->tail =0;
    mailbox->size = buffer_size;
    mailbox->message_count =0;

// 3. 初始化状态标志
    mailbox->flags = MAILBOX_FLAG_READY | MAILBOX_FLAG_EMPTY;

// 4. 初始化访问锁
spinlock_init(&mailbox->lock);

// 5. 设置所有者核心
    mailbox->owner_core = core_id;

// 6. 清零缓冲区
memset(mailbox->buffer,0, buffer_size);

MAILBOX_LOG("Mailbox initialized successfully");
return MAILBOX_OK;
}

/**
 * 发送消息到邮箱
 */
mailbox_result_tmailbox_send(mailbox_t*mailbox,constvoid*data,
uint32_t length,uint32_t timeout_ms){
mailbox_message_t message;
uint64_t start_time;
uint32_t retry_count =0;

// 参数检查
if(mailbox ==NULL|| data ==NULL){
MAILBOX_LOG("Invalid parameters");
return MAILBOX_INVALID_PARAM;
}

if(length ==0|| length > MAX_MESSAGE_SIZE){
MAILBOX_LOG("Invalid message length: %d", length);
return MAILBOX_INVALID_SIZE;
}

// 检查邮箱状态
if((mailbox->flags & MAILBOX_FLAG_READY)==0){
MAILBOX_LOG("Mailbox is not ready");
return MAILBOX_NOT_READY;
}

if(mailbox->owner_core ==get_cpu_id()){
MAILBOX_LOG("Cannot send to own mailbox");
return MAILBOX_SELF_SEND_ERROR;
}

// 准备消息
    message.header.magic = MAILBOX_MAGIC_NUMBER;
    message.header.length = length;
    message.header.sequence =get_next_message_sequence();
    message.header.source_core =get_cpu_id();
    message.header.timestamp =get_system_timestamp();
    message.header.checksum =0;

// 复制数据
if(length >0){
memcpy(message.data, data, length);
}

// 计算校验和
    message.header.checksum =calculate_checksum(&message,
sizeof(mailbox_message_header_t)+ length);

MAILBOX_LOG("Preparing to send message, seq=%d, length=%d",
               message.header.sequence, length);

// 尝试发送（支持重试）
    start_time =get_system_timestamp();

while(retry_count < MAX_SEND_RETRY_COUNT){
mailbox_result_t result =try_send_message(mailbox,&message,
sizeof(mailbox_message_header_t)+ length);

if(result == MAILBOX_OK){
// 发送成功，触发中断通知接收方
ipi_send(mailbox->owner_core, IPI_TYPE_MAILBOX,
                    MAILBOX_EVENT_MESSAGE_ARRIVED, IPI_PRIORITY_NORMAL);

MAILBOX_LOG("Message sent successfully, seq=%d", message.header.sequence);
return MAILBOX_OK;
}

// 发送失败，检查是否超时
uint64_t current_time =get_system_timestamp();
if(timeout_ms >0&&(current_time - start_time)>(timeout_ms *1000)){
MAILBOX_LOG("Send timeout after %d ms", timeout_ms);
return MAILBOX_TIMEOUT;
}

// 等待后重试
        retry_count++;
uint32_t delay_us = SEND_RETRY_DELAY *(1<< retry_count);// 指数退避
        delay_us =MIN(delay_us, MAX_RETRY_DELAY);

MAILBOX_LOG("Send failed, retry %d after %d us delay", retry_count, delay_us);
delay_us(delay_us);
}

MAILBOX_LOG("Failed to send message after %d retries", MAX_SEND_RETRY_COUNT);
return MAILBOX_SEND_FAILED;
}

/**
 * 尝试发送消息（内部函数）
 */
staticmailbox_result_ttry_send_message(mailbox_t*mailbox,
constmailbox_message_t*message,
uint32_t total_length){
uint32_t free_space, write_pos, wrap_length;

// 获取锁
if(!spinlock_trylock(&mailbox->lock)){
return MAILBOX_BUSY;
}

// 计算可用空间
if(mailbox->head >= mailbox->tail){
        free_space = mailbox->size - mailbox->head + mailbox->tail;
}else{
        free_space = mailbox->tail - mailbox->head;
}

// 检查空间是否足够
if(free_space < total_length +sizeof(uint32_t)){// 预留长度字段空间
spinlock_unlock(&mailbox->lock);
return MAILBOX_FULL;
}

// 写入消息长度（用于接收方快速确定消息边界）
    write_pos = mailbox->head;
*((uint32_t*)(mailbox->buffer + write_pos))= total_length;
    write_pos =(write_pos +sizeof(uint32_t))% mailbox->size;

// 写入消息数据
if(write_pos + total_length <= mailbox->size){
// 不跨越边界
memcpy(mailbox->buffer + write_pos, message, total_length);
        write_pos += total_length;
}else{
// 跨越边界，需要分两次拷贝
        wrap_length = mailbox->size - write_pos;
memcpy(mailbox->buffer + write_pos, message, wrap_length);
memcpy(mailbox->buffer,(uint8_t*)message + wrap_length, total_length - wrap_length);
        write_pos = total_length - wrap_length;
}

// 更新写指针
    mailbox->head = write_pos;
    mailbox->message_count++;

// 更新状态标志
    mailbox->flags &=~MAILBOX_FLAG_EMPTY;
if(mailbox->message_count >= MAILBOX_HIGH_WATERMARK){
        mailbox->flags |= MAILBOX_FLAG_NEAR_FULL;
}

// 释放锁
spinlock_unlock(&mailbox->lock);

return MAILBOX_OK;
}

/**
 * 从邮箱接收消息
 */
mailbox_result_tmailbox_receive(mailbox_t*mailbox,void*buffer,
uint32_t buffer_size,uint32_t*received_length,
uint32_t timeout_ms){
mailbox_message_t message;
uint32_t message_length, read_pos, wrap_length;
uint64_t start_time;

// 参数检查
if(mailbox ==NULL|| buffer ==NULL|| received_length ==NULL){
MAILBOX_LOG("Invalid parameters");
return MAILBOX_INVALID_PARAM;
}

// 检查邮箱状态
if((mailbox->flags & MAILBOX_FLAG_READY)==0){
MAILBOX_LOG("Mailbox is not ready");
return MAILBOX_NOT_READY;
}

// 检查是否是邮箱所有者
if(mailbox->owner_core !=get_cpu_id()){
MAILBOX_LOG("Not the owner of this mailbox");
return MAILBOX_NOT_OWNER;
}

MAILBOX_LOG("Waiting for message, timeout=%d ms", timeout_ms);

// 等待消息到达
    start_time =get_system_timestamp();

while(1){
// 检查是否有消息
if(mailbox->message_count >0){
break;
}

// 检查超时
if(timeout_ms >0){
uint64_t current_time =get_system_timestamp();
if(current_time - start_time >(timeout_ms *1000)){
MAILBOX_LOG("Receive timeout");
return MAILBOX_TIMEOUT;
}
}

// 短暂等待
delay_us(100);
}

// 获取锁
spinlock_lock(&mailbox->lock);

// 读取消息长度
    read_pos = mailbox->tail;
    message_length =*((uint32_t*)(mailbox->buffer + read_pos));
    read_pos =(read_pos +sizeof(uint32_t))% mailbox->size;

// 检查消息长度是否有效
if(message_length ==0|| message_length > MAX_MESSAGE_SIZE +sizeof(mailbox_message_header_t)){
MAILBOX_LOG("Invalid message length: %d", message_length);
spinlock_unlock(&mailbox->lock);
return MAILBOX_CORRUPTED;
}

// 检查缓冲区是否足够
if(buffer_size < message_length -sizeof(mailbox_message_header_t)){
MAILBOX_LOG("Buffer too small: need %d, have %d",
                   message_length -sizeof(mailbox_message_header_t), buffer_size);
spinlock_unlock(&mailbox->lock);
return MAILBOX_BUFFER_TOO_SMALL;
}

// 读取消息数据
if(read_pos + message_length <= mailbox->size){
// 不跨越边界
memcpy(&message, mailbox->buffer + read_pos, message_length);
        read_pos += message_length;
}else{
// 跨越边界，需要分两次拷贝
        wrap_length = mailbox->size - read_pos;
memcpy(&message, mailbox->buffer + read_pos, wrap_length);
memcpy((uint8_t*)&message + wrap_length, mailbox->buffer,
               message_length - wrap_length);
        read_pos = message_length - wrap_length;
}

// 验证消息
if(message.header.magic != MAILBOX_MAGIC_NUMBER){
MAILBOX_LOG("Invalid message magic: 0x%08X", message.header.magic);
spinlock_unlock(&mailbox->lock);
return MAILBOX_CORRUPTED;
}

uint32_t calculated_checksum =calculate_checksum(&message, message_length);
if(calculated_checksum != message.header.checksum){
MAILBOX_LOG("Checksum mismatch: expected 0x%08X, got 0x%08X",
                   message.header.checksum, calculated_checksum);
spinlock_unlock(&mailbox->lock);
return MAILBOX_CORRUPTED;
}

// 复制数据到用户缓冲区
*received_length = message.header.length;
if(message.header.length >0){
memcpy(buffer, message.data, message.header.length);
}

// 更新读指针
    mailbox->tail = read_pos;
    mailbox->message_count--;

// 更新状态标志
if(mailbox->message_count ==0){
        mailbox->flags |= MAILBOX_FLAG_EMPTY;
}

if(mailbox->message_count < MAILBOX_LOW_WATERMARK){
        mailbox->flags &=~MAILBOX_FLAG_NEAR_FULL;
}

// 释放锁
spinlock_unlock(&mailbox->lock);

MAILBOX_LOG("Message received successfully, seq=%d, length=%d, from core=%d",
               message.header.sequence, message.header.length, message.header.source_core);

return MAILBOX_OK;
}
